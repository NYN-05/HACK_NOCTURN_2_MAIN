# Frontend + UI/UX Guide

This guide is a practical path for becoming a strong front-end developer with a serious UI/UX focus. It prioritizes free, structured, high-quality resources first, then adds selective paid or optional resources for practice and speed.[cite:27][cite:29]

## Best starting point

The strongest free foundation is MDN's curriculum and learning area. MDN describes its curriculum as beginner-level, self-paced, free, and built around the essential skills needed for front-end career success, while its learning area provides a structured tutorial path designed to move learners from beginner to comfortable.[cite:35][cite:32]

Start in this order:

1. MDN Getting started
2. MDN Core modules
3. MDN Extensions
4. Then add React, TypeScript, testing, performance, and portfolio work

## What to learn in order

A modern front-end path still begins with HTML, CSS, and JavaScript before frameworks. MDN explicitly recommends learning vanilla HTML, CSS, and JavaScript before jumping into frameworks, and its path emphasizes forms, testing, accessibility, and modern tooling as part of professional front-end development.[cite:29][cite:32]

A practical 8-month roadmap published in 2026 follows the same pattern: foundations first, then React and TypeScript, then production skills, then job readiness through deployed projects and interview prep.[cite:30]

| Stage | Focus | Main outcome |
|---|---|---|
| 1 | HTML, CSS, responsive design | Build clean static pages |
| 2 | JavaScript, DOM, APIs | Build interactive apps |
| 3 | Accessibility, forms, performance | Build usable and fast products |
| 4 | React, TypeScript, Git | Build modern maintainable apps |
| 5 | Testing, deployment, portfolio | Become job-ready |
| 6 | UI/UX thinking, design systems, polish | Stand out from typical front-end developers |

## UI/UX focus

If you want to stand out, treat UI/UX as a parallel track, not a separate career later. Current front-end roadmaps increasingly include UI/UX fundamentals, component architecture, accessibility, and clean interaction design as part of modern engineering practice.[cite:27][cite:33][cite:36]

For quick UI/UX fundamentals, Zero To Mastery's UI/UX cheat sheet is a helpful summary resource, although the PDF version requires email signup. It is best used as revision material rather than as your main course.[cite:28]

## Best weekly plan

Use a build-first schedule:

- **Day 1-2:** MDN HTML/CSS modules, recreate one clean landing page.[cite:32]
- **Day 3-4:** JavaScript fundamentals, DOM manipulation, forms, and API basics.[cite:29][cite:32]
- **Day 5:** UI/UX study, analyze spacing, hierarchy, color, typography, and flows from a good product.[cite:28][cite:31]
- **Day 6:** Build one polished mini-project with responsiveness and accessibility.[cite:29][cite:32]
- **Day 7:** Review, refactor, deploy, and write what improved usability and code quality.[cite:30][cite:36]

## Projects that make you better

The best projects are not only functional but also visually thoughtful and usable. Roadmaps aimed at employability consistently point toward responsive sites, API-based apps, tested apps, and deployed portfolio projects as meaningful milestones.[cite:30][cite:36]

Build these in order:

1. Responsive portfolio website
2. Pricing page plus dark mode toggle
3. Todo or notes app with filters and local storage
4. Weather or movie app using an API
5. Dashboard with charts and reusable components
6. E-commerce UI with cart interactions
7. Full React + TypeScript project with deployment

## How to become exceptional

The jump from average to excellent usually comes from five habits:

- Build from scratch before using templates.[cite:29][cite:32]
- Study accessibility early, not at the end.[cite:29][cite:32]
- Recreate strong product UIs to learn spacing and hierarchy.[cite:28][cite:33]
- Write cleaner components and maintainable styling systems.[cite:27][cite:33]
- Ship projects publicly and iterate based on feedback.[cite:30][cite:35]

## Recommended path for you

Since you already have programming experience and are comfortable with development tools, the best route is not a slow beginner-only path but a structured build loop: learn fundamentals from MDN, practice UI breakdowns weekly, then move into React + TypeScript once your HTML/CSS/JavaScript base is solid.[cite:1][cite:32][cite:35]

A strong target for the next 4 to 6 months is to finish the MDN core path, complete at least 4 polished projects, learn basic UX principles, and ship one serious React project that shows both engineering and interface quality.[cite:30][cite:32][cite:36]
